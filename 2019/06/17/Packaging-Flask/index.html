Packaging Flask or Python Project

There are some advantages if you publish and install a project in one packet, which you won’t concern about the dependencies and environment settings, all you need to do is just typing a command in the terminal or double clicking the installer.

Packaging Python Projects you can see here, and you will know how to create package. And you should to create a pypi.org account. If you use this way, people can install your project like pip install package-name.

Or, just for your personal reason, all you need to do is following steps below. This method is based on Packaging Flask project.

Prepare installer

In root folder of project, create somethings like these: setup.py, MANIFEST.in and project files.

setuptools is built in python, updating it to the newest one is perfect. In setup.py, name should be the same with what you did in virtural environment: export FLASK_APP=hello.py, so, it could be like name='hello.py'. packages tells Python what package directories (and the Python files they contain) to include. find_packages() finds these directories automatically so you don’t have to type them out. To include other files, such as the static and templates directories, include_package_data is set. Python needs another file named MANIFEST.in to tell what this other data is.

We can use pip freeze or pipreqs to generate requirements.txt to show what we need in the project. Since pip freeze will output all the requires in the OS environment, we’d better use pip install pipreqs to install pipreqs and then just use it to generate dependences in the project. After we got pipreqs, pipreqs ./ can write requirements to requirements.txt in ./ folder. This could trigger a issue:

Just go to pipreqs.py and modify encoding to ‘utf-8’ in line 75.

This tells Python to copy everything in the static and templates directories, and the schema.sql file, but to exclude all bytecode files.

Install

Copy the whole project to the work machine, and the create a virtual environment. Use pip to install your project in the virtual environment.

This tells pip to find setup.py in the current directory and install it. And it outputs all the installments and steps.
In the new environment, do export FLASK_APP=hello.py and export FLASK_ENV=development if you need again. Type flask run, the project will run ~
