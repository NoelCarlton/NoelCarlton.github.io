3、挖坑与填坑

Question:

Error: Could not locate a Flask application. You did not provide the “FLASK_APP” environment variable, and a “wsgi.py” or “app.py” module was not found in the current directory.

Answer:

这是在Windows power shell 中创建环境变量语法错误导致，在powershell中设置环境变量的命令是：$env:FLASK_APP=”app”，
其它终端中输入命令的方法：

Question:

> flask run
* Serving Flask-SocketIO app "app"
* Forcing debug mode off
Usage: flask run [OPTIONS]

Error: Could not import "code.app".

Answer：

项目之前的目录结构和本文发布的时候，目录结构有所不同：报此错的时候，app同级目录有个app.py作为程序启动入口。当中app文件夹中创建了init.py这python认为改文件夹是一个包，将app.py作为启动文件则认为没必要将其放到包中要么设置app目录为环境信息，把app.py删除，app中添加init.py,要么删除init.py，在app同级目录创建app.py，然后在app文件夹同级目录使用export FLASK_APP=app.py设置环境
